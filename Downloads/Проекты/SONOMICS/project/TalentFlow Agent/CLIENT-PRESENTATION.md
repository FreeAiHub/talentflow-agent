## 🎬 Как это работает: 4 простых шага

### Шаг 1: 🔍 Автоматический мониторинг вакансий

```
┌─────────────────────────────────────────────────────┐
│           Источники вакансий                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│  │ Djinni   │  │ Work.ua  │  │ LinkedIn │          │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘          │
│       │             │              │                 │
│       └─────────────┼──────────────┘                 │
│                     ▼                                │
│            ┌────────────────┐                        │
│            │  TalentFlow    │                        │
│            │    Parser      │                        │
│            └────────────────┘                        │
└─────────────────────────────────────────────────────┘

✅ Сканирование каждые 5-10 минут
✅ Фильтрация по критериям (стек, уровень, гео)
✅ Дедупликация (исключение дубликатов)
```

**Пример:**
- Система находит новую вакансию "Senior Python Developer" на Djinni
- Автоматически проверяет: соответствует ли критериям?
- Если да → переходит к анализу
- Если нет → пропускает

---

### Шаг 2: 🤖 AI-анализ вакансии (через Flowise)

```
┌─────────────────────────────────────────────────────┐
│                 AI-Анализ                           │
│                                                      │
│  Текст вакансии                                     │
│       ↓                                              │
│  ┌──────────────────────────────┐                   │
│  │   Flowise Analyzer           │                   │
│  │   (GPT-4o / OpenRouter)      │                   │
│  └──────────────────────────────┘                   │
│       ↓                                              │
│  Структурированный анализ:                          │
│  ├─ Pain Points (боли компании)                     │
│  ├─ KPIs (метрики успеха)                           │
│  ├─ Required Skills (технологии)                    │
│  ├─ Seniority Level (уровень)                       │
│  └─ Urgency Score (приоритет)                       │
└─────────────────────────────────────────────────────┘
```

**Что извлекает AI:**

**Входные данные:**
```
Senior Python Developer нужен для оптимизации backend 
нашего финтех-продукта. У нас 500K+ пользователей, 
но сервис тормозит при пиковых нагрузках. Нужен опыт 
с FastAPI, PostgreSQL, Redis, Docker.
```

**Выходные данные (JSON):**
```json
{
  "pain_points": [
    "Проблемы с производительностью при высоких нагрузках",
    "Отсутствие экспертизы в оптимизации backend",
    "Необходимость масштабирования под рост пользователей"
  ],
  "kpis": [
    "Снизить latency на 50%",
    "Увеличить throughput в 3 раза",
    "Обеспечить 99.9% uptime"
  ],
  "required_skills": ["Python", "FastAPI", "PostgreSQL", "Redis", "Docker"],
  "seniority_level": "Senior",
  "urgency_score": 8
}
```

### Шаг 3: ✨ Генерация персонализированного отклика

```
┌─────────────────────────────────────────────────────┐
│         Персонализация через RAG                    │
│                                                      │
│  Результаты анализа                                 │
│       ↓                                              │
│  ┌──────────────────────────────┐                   │
│  │  Vector DB (Pinecone)         │                   │
│  │  База успешных откликов       │                   │
│  └──────────────┬───────────────┘                   │
│                 │ Похожие кейсы                      │
│                 ↓                                     │
│  ┌──────────────────────────────┐                   │
│  │   Flowise Generator           │                   │
│  │   (OpenRouter)                │                   │
│  └──────────────────────────────┘                   │
│       ↓                                              │
│  Готовый персонализированный отклик                 │
└─────────────────────────────────────────────────────┘
```

**Пример сгенерированного отклика:**

```
Вижу, что у вас серьезная задача по оптимизации backend 
под 500K+ пользователей. Знакомая история — в прошлом 
проекте (EdTech платформа с 2M пользователей) решал 
похожую проблему.

Что сделал:
• Рефакторинг API на FastAPI → latency упала с 800ms до 120ms
• Оптимизация PostgreSQL индексов → throughput вырос в 4 раза
• Внедрение Redis caching → снижение нагрузки на БД на 60%

Результат: система выдержала пиковую нагрузку 50K RPS 
при 99.95% uptime.

Готов показать архитектурное решение для вашего кейса. 
Предлагаю 30-минутный созвон на этой неделе?

Календарь: https://calendly.com/talentflow/demo
```

**Почему это работает:**
- ✅ Демонстрирует понимание проблемы
- ✅ Конкретные цифры и метрики
- ✅ Готовое решение, а не "я могу научиться"
- ✅ Четкий CTA (Call-to-Action)

### Шаг 4: 📤 Автоматическая отправка и лид-менеджмент

```
┌─────────────────────────────────────────────────────┐
│              Автоматизация отправки                 │
│                                                      │
│  Готовый отклик                                     │
│       ↓                                              │
│  ┌──────────────────┐    ┌──────────────────┐      │
│  │  Отправка на     │    │  Создание лида   │      │
│  │  платформу       │───▶│  в CRM/Linear    │      │
│  └──────────────────┘    └──────────────────┘      │
│       ↓                           ↓                  │
│  ┌──────────────────┐    ┌──────────────────┐      │
│  │  Планирование    │    │  Уведомление     │      │
│  │  встречи         │───▶│  менеджера       │      │
│  │  (Calendly)      │    │  (Slack/Email)   │      │
│  └──────────────────┘    └──────────────────┘      │
└─────────────────────────────────────────────────────┘
```

**Что происходит автоматически:**

1. ✉️ Отклик отправляется рекрутеру на платформе
2. 📅 Вставляется ссылка на Calendly для бронирования встречи
3. 💼 Создается запись в CRM с полным контекстом
4. 🔔 Менеджер получает уведомление о новом теплом лиде
5. 📊 Обновляется статистика и метрики
